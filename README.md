# Удалённая IT поддержка внутренних пользователей

## Запуск

1. Файл `.env` уже подготовлен (или скопируйте из `.env.example`).
2. `keystore.p12` лежит в `src/main/resources/` (для локального HTTPS).
3. `docker compose up --build` — PostgreSQL и приложение на `https://localhost:8443`.

## Секреты и CI

В GitHub Actions настроены `KEYSTORE_BASE64` и `KEYSTORE_PASSWORD`.  
Локальные переменные — в `.env`.

## Тема
Сервис для отслеживания и обработки IT-тикетов внутри компании.

## Основные сущности
- Ticket — тикет (инцидент, запрос или проблема)
- User — пользователь, создавший тикет
- Executor — сотрудник, который выполняет тикет
- Category — категория тикета
- SLA — время реакции и решения

## Операции
- CRUD для всех сущностей
- Назначение тикета исполнителю
- Закрытие тикета с указанием решения
- Получение просроченных тикетов
- Эскалация просроченных тикетов
- Получение тикетов конкретного исполнителя
